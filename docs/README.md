## Documentation Standards

When adding new documentation:

1. **Location**: Place in appropriate directory
   - Root: Essential project documentation
   - `docs/`: Additional guides and HTML demos
   - `docs/archive/`: Historical documentation only

2. **Format**: Use Markdown (.md) for text documentation

3. **Naming**: Use descriptive, uppercase names with underscores
   - Example: `INTEGRATION_GUIDE.md`, `API_REFERENCE.md`

4. **Content**: Include
   - Clear title and purpose
   - Table of contents for long documents
   - Code examples with syntax highlighting
   - Links to related documentation

5. **Updates**: Keep documentation synchronized with code changes
